1. EXECUTIVE SUMMARY
1.1 Product Overview
Gemini Code Analyzer is a full-stack web application that leverages Google's Gemini 2.5 Flash AI model to automatically analyze source code, detect errors, provide improvement suggestions, and generate optimized code versions across multiple programming languages. The system combines AI-powered analysis with robust database management to provide a comprehensive code quality assessment tool.
1.2 Business Objectives
•	Reduce code review time by automating initial error detection
•	Improve code quality through AI-powered suggestions
•	Create a centralized repository of code analysis history
•	Provide educational value for developers learning best practices
•	Demonstrate integration of modern technologies (AI, databases, web services)
1.3 Target Users
•	Primary: Individual developers, students, and coding learners
•	Secondary: Small development teams, educational institutions, code review teams
•	Tertiary: Technical recruiters evaluating coding skills
1.4 Key Success Metrics
•	Analysis accuracy rate: >85%
•	Response time: <5 seconds per analysis
•	System uptime: >95%
•	User satisfaction: 4+/5 rating
•	Database query performance: <100ms for history retrieval

2.1.2 Database Management
Feature ID: F-002
Priority: Critical
Description: Persistent storage of analysis results in MySQL database
Requirements:
•	FR-002.1: System shall save all analysis results to MySQL database automatically
•	FR-002.2: Database schema shall include 10 fields: id, original_code, timestamp, language, has_errors, errors, output, suggestions, optimal_code, created_at
•	FR-002.3: System shall use prepared statements to prevent SQL injection
•	FR-002.4: System shall handle database connection failures gracefully
•	FR-002.5: System shall support concurrent database operations (minimum 10 connections)
•	FR-002.6: Database queries shall complete within 100ms for standard operations
•	FR-002.7: System shall maintain referential integrity and data consistency
Database Schema:
CREATE TABLE code_analysis (
    id INT AUTO_INCREMENT PRIMARY KEY,
    original_code TEXT NOT NULL,
    timestamp DATETIME NOT NULL,
    language VARCHAR(100),
    has_errors BOOLEAN DEFAULT FALSE,
    errors TEXT,
    output TEXT,
    suggestions TEXT,
    optimal_code TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
Indexes:
•	Primary key on id
•	Index on timestamp for chronological queries
•	Index on language for filtering
•	Index on has_errors for statistics

4.3 Data Flow
Analysis Flow:
1. User inputs code in frontend textarea
2. User clicks "Analyze Code" button
3. Frontend sends POST request to /api/analyze with code
4. CodeAnalyzerServer receives request
5. Server calls GeminiCodeAnalyzer.analyzeCodeStatic()
6. GeminiCodeAnalyzer sends HTTP request to Gemini API
7. Gemini API analyzes code and returns JSON response
8. GeminiCodeAnalyzer parses response into CodeAnalysisResult object
9. Server calls DatabaseManager.saveToDatabase()
10. DatabaseManager inserts record into MySQL database
11. Server returns JSON response to frontend with result + save status
12. Frontend displays results in tabbed interface
13. Frontend calls /api/history to refresh history list
14. Frontend calls /api/stats to update statistics
History Retrieval Flow:
1. Frontend calls GET /api/history?limit=10
2. CodeAnalyzerServer routes to getHistory()
3. getHistory() calls DatabaseManager.getRecentHistory(10)
4. DatabaseManager executes SQL query with LIMIT
5. Database returns ResultSet with recent records
6. DatabaseManager converts ResultSet to List<HistoryItem>
7. Server returns JSON array to frontend
8. Frontend renders history items with badges and previews
History Item Load Flow:
1. User clicks on history item
2. Frontend calls GET /api/analysis/:id
3. Server calls DatabaseManager.getAnalysisById(id)
4. DatabaseManager queries database for specific record
5. Database returns single record
6. DatabaseManager converts to CodeAnalysisResult
7. Server returns full analysis result to frontend
8. Frontend populates code textarea and displays results

6. ERROR HANDLING
6.1 Frontend Errors
Error Type	User Message	Action
Empty code input	"Please enter some code to analyze"	Red toast notification, focus textarea
Server offline	"Server Offline - Start the backend server"	Red status indicator, disable analyze button
API timeout	"Request timed out. Please try again."	Red toast, retry button
Network error	"Network error. Check your connection."	Red toast, retry button
Invalid JSON response	"Unexpected response from server"	Console log error, show generic message
6.2 Backend Errors
Error Type	HTTP Status	Response	Logging
Empty code	400	{"error": "Code cannot be empty"}	WARN level
Gemini API failure	500	{"error": "Analysis failed: <message>"}	ERROR level with stack trace
Database connection	500	{"error": "Database unavailable"}	ERROR level
Invalid analysis ID	404	{"error": "Analysis not found"}	INFO level
SQL exception	500	{"error": "Database error occurred"}	ERROR level with query details
6.3 Database Errors
Error Type	Handling Strategy	Recovery
Connection timeout	Retry 3 times with 2s delay	Log and notify user
Constraint violation	Roll back transaction	Return error to user
Disk space full	Stop write operations	Alert admin, show read-only mode
Deadlock	Retry transaction	Log occurrence

17.1 Input Validation
Backend Validation:
public class SecurityUtils {
    
    // Sanitize user input
    public static String sanitizeInput(String input) {
        if (input == null) return "";
        
        // Remove potentially dangerous characters
        return input.replaceAll("[<>\"'%;()&+]", "")
                    .trim();
    }
    
    // Validate code length
    public static boolean isValidCodeLength(String code) {
        return code != null && 
               code.length() >= 10 && 
               code.length() <= 10000;
    }
    
    // Check for SQL injection patterns
    public static boolean containsSQLInjection(String input) {
        String[] sqlKeywords = {"DROP", "DELETE", "INSERT", "UPDATE", 
                               "CREATE", "ALTER", "EXEC", "UNION"};
        String upperInput = input.toUpperCase();
        
        for (String keyword : sqlKeywords) {
            if (upperInput.contains(keyword)) {
                return true;
            }
        }
        return false;
    }
}
Frontend Validation:
function validateCode(code) {
    // Check length
    if (code.length < 10) {
        return { valid: false, error: "Code must be at least 10 characters" };
    }
    
    if (code.length > 10000) {
        return { valid: false, error: "Code exceeds maximum length" };
    }
    
    // Check for dangerous patterns
    const dangerousPatterns = [
        /<script>/gi,
        /javascript:/gi,
        /on\w+\s*=/gi  // Event handlers like onclick=
    ];
    
    for (let pattern of dangerousPatterns) {
        if (pattern.test(code)) {
            return { valid: false, error: "Code contains potentially dangerous content" };
        }
    }
    
    return { valid: true };
}

WHERE timestamp < DATE_SUB(NOW(), INTERVAL 1 YEAR);

-- Archive before deleting (recommended)
CREATE TABLE code_analysis_archive AS 
SELECT * FROM code_analysis 
WHERE timestamp < DATE_SUB(NOW(), INTERVAL 1 YEAR);

-- Then delete from main table
DELETE FROM code_analysis 
WHERE timestamp < DATE_SUB(NOW(), INTERVAL 1 YEAR);
Monitor Database Size:
SELECT 
    table_name AS 'Table',
    ROUND(((data_length + index_length) / 1024 / 1024), 2) AS 'Size (MB)'
FROM information_schema.TABLES 
WHERE table_schema = 'code_analyzer'
ORDER BY (data_length + index_length) DESC;
